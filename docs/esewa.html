eSewa Integration

eSewa is Nepal's first and largest payment service provider. Unlike Khalti, eSewa uses a form-submission model — your backend builds a signed payload, and the frontend POSTs directly to eSewa.

⚠️ Critical: Sandbox vs Production The most common bug — product-code is EPAYTEST in sandbox but YOUR merchant code in production. Always set this via environment variable.

How eSewa Works

Your Backend ──▶ builds signed EsewaFormPayload
                         │
               returns to your frontend
                         │
       Frontend POSTs form to eSewa URL
                         │
               User pays on eSewa
                         │
  eSewa redirects to successUrl?data=BASE64_JSON
                         │
         ⚠️  DO NOT trust redirect alone!
                         │
Your Backend: decode + verify HMAC signature
                         │
Your Backend: call eSewa Status API
                         │
                 ✅ mark as paid

Step 1 — Build Form Payload

@PostMapping("/api/payment/esewa/initiate")
public ResponseEntity<EsewaFormPayload> initiatePayment(
        @RequestBody PaymentRequest request) {

    // Generate unique UUID — save this to DB!
    String uuid = EsewaClient.generateTransactionUuid();
    orderRepo.saveUuid(request.orderId(), uuid);

    // ⚠️ eSewa uses NPR directly — NOT paisa like Khalti
    EsewaFormPayload payload = esewaClient.buildFormPayload(
        request.amountNPR(),   // BigDecimal — e.g. new BigDecimal("100.00")
        uuid
    );

    return ResponseEntity.ok(payload);
    // Frontend POSTs this form to payload.formActionUrl()
}

Step 2 — Frontend Form Submission

// Angular/TypeScript example
function submitEsewaForm(payload: EsewaFormPayload): void {
    const form = document.createElement('form');
    form.method = 'POST';
    form.action = payload.form_action_url;

    const fields = [
        'amount', 'tax_amount', 'total_amount',
        'transaction_uuid', 'product_code',
        'product_service_charge', 'product_delivery_charge',
        'success_url', 'failure_url',
        'signed_field_names', 'signature'
    ];

    fields.forEach(field => {
        const input = document.createElement('input');
        input.type  = 'hidden';
        input.name  = field;
        input.value = (payload as any)[field];
        form.appendChild(input);
    });

    document.body.appendChild(form);
    form.submit();
}

Step 3 — Verify Callback

@GetMapping("/api/payment/esewa/callback")
public ResponseEntity<String> handleCallback(
        @RequestParam String data) {

    // verifyCallback() does 3 things automatically:
    // 1. Decode Base64 → EsewaCallbackData
    // 2. Verify HMAC-SHA256 signature (tamper protection)
    // 3. Call eSewa Status API for final confirmation
    EsewaVerificationResult result = esewaClient.verifyCallback(data);

    if (!result.isPaymentSuccessful()) {
        return ResponseEntity.badRequest()
            .body("Not confirmed: " + result.statusResponse().status());
    }

    String uuid = result.callbackData().transactionUuid();
    orderRepo.markAsPaid(uuid, result.statusResponse().refId());
    return ResponseEntity.ok("Payment successful!");
}

Amount — Important Note

ℹ️ eSewa uses NPR directly (not paisa like Khalti) NPR 100 → send new BigDecimal("100.00"). Always use 2 decimal places.
