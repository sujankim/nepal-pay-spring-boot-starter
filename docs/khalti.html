How Khalti Works

ℹ️

Khalti uses an API-first model — your backend calls Khalti's API directly to initiate a payment, then verifies the result server-side after the user returns.

Your Backend ─── POST ───▶ Khalti API (/epayment/initiate/) ◀─── { pidx, paymentUrl } ─── │ save pidx to DB ← IMPORTANT │ redirect user to paymentUrl │ User pays on Khalti │ Khalti redirects to your returnUrl (?pidx=xxx&status=Completed) │ ⚠️ NEVER trust redirect params alone! │ Your Backend ─── POST ───▶ Khalti API (/epayment/lookup/) ◀─── { status: "Completed" } ─── │ ✅ mark order as paid

initiatePayment()

KhaltiInitiateResponse res = khaltiClient.initiatePayment(
    KhaltiInitiateRequest.builder()
        .amount(10000L)               // ⚠️ PAISA not NPR — NPR 100 = 10000
        .purchaseOrderId("ORD-001")   // your unique order ID
        .purchaseOrderName("Pro Plan") // shown on Khalti payment page
        .returnUrl("https://yourapp.com/khalti/callback")  // optional override
        .customerInfo(                 // optional — shown in Khalti dashboard
            "Sujan Lamichhane",
            "sujan@example.com",
            "9800000001"
        )
        .build()
);

// IMPORTANT: save pidx to DB BEFORE redirecting user
orderRepo.savePidx(orderId, res.pidx());

// Redirect user to this URL
return res.paymentUrl();
⚠️

Amount is in Paisa, not NPR. NPR 100 → send 10000L. Minimum payment is NPR 10 = 1000L paisa.

Request fields

Field Required Type Description
amount Required long Amount in paisa (NPR × 100)
purchaseOrderId Required String Your unique order identifier
purchaseOrderName Required String Human-readable name shown on payment page
returnUrl Optional String Overrides nepalpay.khalti.return-url
websiteUrl Optional String Overrides nepalpay.khalti.website-url
customerInfo Optional CustomerInfo Name, email, phone — shown in dashboard

lookupPayment()

Called at your returnUrl after the user completes payment. Always call this — never trust the redirect parameters alone. 

// Called at your returnUrl after user completes payment
@GetMapping("/khalti/callback")
public ResponseEntity<String> callback(@RequestParam String pidx) {

    // ✅ ALWAYS verify server-side — redirect params can be faked
    KhaltiLookupResponse lookup = khaltiClient.lookupPayment(pidx);

    if (!lookup.isPaymentSuccessful()) {
        return ResponseEntity.badRequest().body("Not confirmed");
    }

    // Load orderId from DB using the saved pidx mapping
    String orderId = orderRepo.findOrderByPidx(pidx);
    orderService.markAsPaid(orderId, lookup.transactionId());

    // ✅ Verify amount was not tampered
    long expectedPaisa = orderRepo.getAmount(pidx);
    if (!lookup.isAmountValid(expectedPaisa)) {
        log.error("Amount mismatch — possible tampering!");
        return ResponseEntity.badRequest().body("Amount mismatch");
    }

    // Safe to mark order as paid
    orderRepo.markPaid(orderId, lookup.transactionId());
    return ResponseEntity.ok("Payment confirmed");
}

Response fields

Field Type Description
pidx String Payment identifier from initiate
status String Raw status string from Khalti API
totalAmount Long Amount paid in paisa — verify this!
transactionId String Khalti's internal transaction ID — needed for refunds
isPaymentSuccessful() boolean true only when status is Completed
isAmountValid(long) boolean Checks totalAmount matches your expected paisa
isRefunded() boolean true when status is Refunded

refundPayment()

⚠️

Refund uses transactionId, NOT pidx. Get transactionId from lookupPayment(pidx).transactionId(). It is only non-null after a payment reaches Completed status.

Full refund

// Step 1: Lookup to get transactionId
KhaltiLookupResponse lookup = khaltiClient.lookupPayment(pidx);

if (!lookup.isPaymentSuccessful()) {
    throw new IllegalStateException("Only completed payments can be refunded");
}

// Step 2: Refund entire amount
KhaltiRefundResponse refund =
    khaltiClient.refundPayment(lookup.transactionId());

if (refund.isRefundSuccessful()) {
    orderRepo.markRefunded(orderId);
}

Partial refund

// Refund NPR 50 (= 5000 paisa) from a completed transaction
KhaltiRefundResponse refund =
    khaltiClient.refundPayment(lookup.transactionId(), 5000L);

if (refund.isRefundSuccessful()) {
    orderRepo.markPartiallyRefunded(orderId, 5000L);
}

Refund rules

RuleDetail
Use transactionId From lookupPayment(pidx).transactionId() — not pidx
Only completed payments Pending / Expired / Canceled → 400 error from Khalti
Partial amount in paisa NPR 50 → 5000L
Full refund refundPayment(transactionId) — no amount arg
🔴

Do not pass pidx to refundPayment(). It expects transactionId. Passing pidx will always fail with a 400 error from Khalti.

Payment Statuses

Status isSuccess() isRefunded() isTerminalFailure() Action
Completed true — — Mark order as paid
REFUNDED — true — Mark order as refunded
PENDING — — — Poll again later
USER_CANCELED — — true Offer retry
CANCELED — — true Offer retry
EXPIRED — — true New payment link
FAILED — — true Offer retry
UNKNOWN — — — Contact Khalti support

A REFUNDED payment was once successful — it is not a failure. Use isRefunded() to detect this case separately from failures.

Security Rules

🔴

Never trust redirect URL parameters alone. Always call lookupPayment(pidx) to confirm.

🔴

Always verify the amount. Call lookup.isAmountValid(expectedPaisa) to detect tampered amounts.

✅

Save pidx to your database before redirecting the user to the payment page.

✅

For refunds, save transactionId from the lookup response to your database. 

// ✅ Correct flow
KhaltiInitiateResponse res = khaltiClient.initiatePayment(request);
orderRepo.savePidx(orderId, res.pidx());   // SAVE FIRST
return res.paymentUrl();                    // THEN redirect

// ✅ Correct verification
KhaltiLookupResponse lookup = khaltiClient.lookupPayment(pidx);
if (!lookup.isPaymentSuccessful()) { /* handle */ }
if (!lookup.isAmountValid(expectedPaisa)) { /* handle tampering */ }
orderRepo.markPaid(orderId, lookup.transactionId());

// ❌ NEVER do this
if (request.getParam("status").equals("Completed")) {
    orderRepo.markPaid(orderId); // DANGEROUS — easily faked
}

Reactive Usage

Use KhaltiReactiveClient from nepal-pay-spring-boot-reactive-starter for non-blocking WebFlux applications.

Initiate payment — reactive

public Mono<String> startKhalti(String orderId, long amountNPR) {
    return khaltiReactiveClient.initiatePayment(
        KhaltiInitiateRequest.builder()
            .amount(amountNPR * 100L)
            .purchaseOrderId(orderId)
            .purchaseOrderName("Pro Plan")
            .build()
    )
    .doOnNext(res -> orderRepo.savePidx(orderId, res.pidx()))
    .map(KhaltiInitiateResponse::paymentUrl);
}

Verify after callback — reactive

@GetMapping("/khalti/callback")
public Mono<ResponseEntity<String>> callback(
        @RequestParam String pidx) {

    return khaltiReactiveClient.lookupPayment(pidx)
    .flatMap(lookup -> {
        if (!lookup.isPaymentSuccessful()) {
            return Mono.just(ResponseEntity.badRequest()
                .body("Not confirmed"));
        }
        String orderId = orderRepo.findOrderByPidx(pidx);
        long expectedPaisa = orderRepo.getAmountPaisa(orderId);
        if (!lookup.isAmountValid(expectedPaisa)) {
            return Mono.just(ResponseEntity.badRequest()
                .body("Amount mismatch — possible tampering"));
        }
        return orderRepo.markPaid(orderId, lookup.transactionId())
            .thenReturn(ResponseEntity.ok("Payment confirmed"));
        });
}

Refund — reactive

// Full refund
public Mono<Boolean> refundKhalti(String transactionId) {
    return khaltiReactiveClient.refundPayment(transactionId)
        .map(KhaltiRefundResponse::isRefundSuccessful);
}

// Partial refund — NPR 50 = 5000 paisa
public Mono<Boolean> partialRefund(String transactionId) {
    return khaltiReactiveClient.refundPayment(transactionId, 5000L)
        .map(KhaltiRefundResponse::isRefundSuccessful);
}
ℹ️

All validation errors in reactive clients are emitted as Mono.error() signals — never thrown directly. Handle them with onErrorResume or StepVerifier.expectError() in tests.
