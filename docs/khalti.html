Khalti Integration

Khalti is Nepal's leading digital wallet and payment gateway. NepalPay provides a type-safe, secure Java client for Khalti ePay v2.

⚠️ Security Rule Always call lookupPayment(pidx) after receiving the redirect. Never trust redirect URL parameters alone — they can be faked.

Payment Flow

Your Backend ──POST──▶ Khalti API ──▶ { pidx, payment_url }
                                                  │
                                        Redirect user to payment_url
                                                  │
                                          User pays on Khalti
                                                  │
                              Khalti redirects to your return_url?pidx=xxx
                                                  │
                        ⚠️  DO NOT trust this redirect alone!
                                                  │
Your Backend ──POST──▶ Khalti Lookup API(pidx) ──▶ { status: "Completed" }
                                                  │
                                        ✅ NOW mark as paid

Configuration

nepalpay:
  khalti:
    secret-key:  ${KHALTI_SECRET_KEY}    # from Khalti merchant dashboard
    return-url:  ${KHALTI_RETURN_URL}    # your callback endpoint
    website-url: ${YOUR_WEBSITE_URL}     # your merchant website
    sandbox: true                        # false for production

Step 1 — Initiate Payment

@PostMapping("/api/payment/khalti/initiate")
public ResponseEntity<Map<String, String>> initiatePayment(
        @RequestBody PaymentRequest request) {

    KhaltiInitiateResponse response = khaltiClient.initiatePayment(
        KhaltiInitiateRequest.builder()
            .amount(request.amountNPR() * 100L) // ⚠️ NPR → paisa (×100)
            .purchaseOrderId(request.orderId())
            .purchaseOrderName(request.productName())
            .build()
    );

    // ✅ Store pidx in DB immediately!
    orderRepo.savePidx(request.orderId(), response.pidx());

    return ResponseEntity.ok(Map.of(
        "pidx",        response.pidx(),
        "payment_url", response.paymentUrl()
    ));
}

Step 2 — Verify Callback

@GetMapping("/api/payment/khalti/callback")
public ResponseEntity<String> handleCallback(
        @RequestParam String pidx) {

    // ✅ Always verify server-side
    KhaltiLookupResponse lookup = khaltiClient.lookupPayment(pidx);

    if (!lookup.isPaymentSuccessful()) {
        return ResponseEntity.badRequest()
            .body("Payment failed: " + lookup.status());
    }

    // ✅ Optional: verify amount was not tampered
    if (!lookup.isAmountValid(expectedAmountPaisa)) {
        throw new SecurityException("Amount mismatch!");
    }

    orderRepo.markAsPaid(lookup.purchaseOrderId(), lookup.transactionId());
    return ResponseEntity.ok("Payment successful!");
}

Payment Statuses

StatusMeaningAction
COMPLETED✅ ConfirmedMark order as paid
PENDING⏳ ProcessingPoll again later
USER_CANCELED❌ User backed outOffer retry
EXPIRED❌ Link expired (60 min)Generate new payment
FAILED❌ FailedOffer retry
UNKNOWN⚠️ UnexpectedContact Khalti

Amount — Important Note

⚠️ Khalti uses PAISA, not NPR NPR 1 = 100 paisa. NPR 100 → send 10000. Minimum: NPR 10 = 1000 paisa. 
// ✅ Correct
long amountPaisa = amountNPR * 100L;

// ❌ Wrong — sending NPR directly to Khalti
.amount(100L)  // This is 1 paisa = NPR 0.01 !
