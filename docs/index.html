🇳🇵 NepalPay Spring Boot Starter

The first production-grade Java library for Nepal payment gateways.
Integrate Khalti and eSewa into any Spring Boot app in under 5 minutes.

Java 21 Spring Boot 4.1.0 51 Tests ✅ MIT License

What is NepalPay?

NepalPay is an open-source Spring Boot Starter that provides a unified, secure, production-grade interface for Nepal's major payment gateways — currently Khalti and eSewa.

Before NepalPay, every Java developer had to write 500+ lines of boilerplate for each gateway, every single project. NepalPay reduces that to 3 lines — with security baked in.

✅ Security by default Server-side payment verification is enforced. The most common vulnerability (trusting redirect URLs) is impossible to make with this library.

Installation

1

Add JitPack repository to pom.xml

<repositories>
    <repository>
        <id>jitpack.io</id>
        <url>https://jitpack.io</url>
    </repository>
</repositories>
2

Add the dependency

<dependency>
    <groupId>com.github.sujankim</groupId>
    <artifactId>nepal-pay-spring-boot-starter</artifactId>
    <version>v0.1.0</version>
</dependency>
3

Configure in application.yml

nepalpay:
  khalti:
    secret-key: ${KHALTI_SECRET_KEY}
    return-url:  ${KHALTI_RETURN_URL}
    website-url: ${YOUR_WEBSITE_URL}
    sandbox: true

  esewa:
    secret-key:   ${ESEWA_SECRET_KEY}
    product-code: ${ESEWA_PRODUCT_CODE}
    success-url:  ${ESEWA_SUCCESS_URL}
    failure-url:  ${ESEWA_FAILURE_URL}
    sandbox: true
4

Inject and use

@Service
@RequiredArgsConstructor
public class PaymentService {

    // Auto-injected — no @Bean, no config class needed
    private final KhaltiClient khaltiClient;
    private final EsewaClient  esewaClient;
}
No @EnableNepalPay needed. Spring Boot auto-configures everything when it detects your keys in application.yml.

Quick Start

Khalti — Initiate and Verify

// Step 1: Initiate payment
KhaltiInitiateResponse response = khaltiClient.initiatePayment(
    KhaltiInitiateRequest.builder()
        .amount(10000L)             // NPR 100 in paisa (×100)
        .purchaseOrderId("ORD-001")
        .purchaseOrderName("Pro Plan")
        .build()
);

// Store pidx — needed for verification!
String pidx = response.pidx();
String redirectUrl = response.paymentUrl();

// Step 2: After redirect, verify server-side
KhaltiLookupResponse lookup = khaltiClient.lookupPayment(pidx);
if (lookup.isPaymentSuccessful()) {
    // ✅ Safe to mark order as paid
}

eSewa — Build Payload and Verify

// Step 1: Build signed form payload
EsewaFormPayload payload = esewaClient.buildFormPayload(

);
