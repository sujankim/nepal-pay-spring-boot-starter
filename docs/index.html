Why NepalPay

Stop writing payment boilerplate

Every project needs payments. None of them should need 500 lines of HMAC signing code — or 200 lines of Reactor pipelines.

⚡

5-Minute Setup

Add dependency → configure keys → inject and use. No @Bean, no config class, no boilerplate at all. Works the same in blocking and reactive apps.

🔄

Reactive (WebFlux)

New in v1.1.0 — Mono<T> responses via WebClient. Full non-blocking pipeline for Spring WebFlux apps. Same YAML config as blocking.

🔐

Secure by Default

HMAC-SHA256, HMAC-SHA512, RSA-SHA256, server-side verification, constant-time signature comparison — all implemented correctly and tested.

🔁

Retry Built In

Exponential backoff with jitter. Blocking uses Thread.sleep() loop. Reactive uses Reactor's Retry.backoff(). Opt-in, configurable per gateway.

💳

Refund Support

Khalti full and partial refunds — both blocking and reactive. refundPayment(transactionId) and refundPayment(transactionId, amountPaisa).

🧪

400+ Tests

MockWebServer HTTP tests, StepVerifier reactive tests, signature correctness tests, retry behavior tests, auto-configuration wiring tests.

New in v1.1.0

Reactive support for Spring WebFlux

The new reactive starter returns Mono<T> via WebClient — fully non-blocking for WebFlux apps. Same YAML, same models, same exceptions.

❌ Blocking in WebFlux — blocks event loop 
// Blocks the Netty thread — bad!
@GetMapping("/pay")
public Mono<String> pay() {
    KhaltiInitiateResponse res =
        khaltiClient.initiatePayment(req);
    return Mono.just(res.paymentUrl());
}
✅ Reactive — fully non-blocking pipeline 
// Fully non-blocking — WebClient all the way
@GetMapping("/pay")
public Mono<String> pay() {
    return khaltiReactiveClient
        .initiatePayment(req)
        .doOnNext(res ->
            orderRepo.savePidx(orderId, res.pidx()))
        .map(KhaltiInitiateResponse::paymentUrl);
}

✅ With NepalPay (blocking) 
// Inject — auto-configured, zero setup
@Service
@RequiredArgsConstructor
public class PaymentService {

    private final KhaltiClient khaltiClient;

    public String initiate(
            String orderId, long nprAmount) {

        var res = khaltiClient.initiatePayment(
            KhaltiInitiateRequest.builder()
                .amount(nprAmount * 100L) // NPR → paisa
                .purchaseOrderId(orderId)
                .purchaseOrderName("Pro Plan")
                .build()
        );

        // Save res.pidx() to DB, redirect to:
        return res.paymentUrl();
    }
}
