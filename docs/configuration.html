Configuration Reference

All properties are under the nepalpay.* prefix. IDE autocomplete is supported — type nepalpay. and your IDE shows all available options.

Khalti Properties

PropertyRequiredDefaultDescription
nepalpay.khalti.secret-key✅—Khalti secret key. Presence of this key enables the KhaltiClient bean.
nepalpay.khalti.return-url✅—URL Khalti redirects to after payment.
nepalpay.khalti.website-url✅—Your merchant website URL.
nepalpay.khalti.sandboxNotruetrue = sandbox / false = production.
nepalpay.khalti.timeout-secondsNo10HTTP timeout for Khalti API calls.

eSewa Properties

PropertyRequiredDefaultDescription
nepalpay.esewa.secret-key✅—eSewa HMAC secret key. Enables EsewaClient bean.
nepalpay.esewa.product-code✅—Merchant code. Sandbox: EPAYTEST. Production: your code.
nepalpay.esewa.success-url✅—URL eSewa redirects to on success.
nepalpay.esewa.failure-url✅—URL eSewa redirects to on failure.
nepalpay.esewa.sandboxNotruetrue = sandbox / false = production.
nepalpay.esewa.timeout-secondsNo10HTTP timeout for eSewa status API calls.

Full Example

# ── Sandbox (development) ─────────────────────────────────────
nepalpay:
  khalti:
    secret-key:  ${KHALTI_SECRET_KEY}
    return-url:  https://yourapp.com/api/payment/khalti/callback
    website-url: https://yourapp.com
    sandbox: true

  esewa:
    secret-key:   ${ESEWA_SECRET_KEY}    # sandbox: 8gBm/:&EnhH.1/q
    product-code: ${ESEWA_PRODUCT_CODE}  # sandbox: EPAYTEST
    success-url:  https://yourapp.com/api/payment/esewa/callback
    failure-url:  https://yourapp.com/payment/failed
    sandbox: true
# ── Production ────────────────────────────────────────────────
nepalpay:
  khalti:
    secret-key:  ${KHALTI_SECRET_KEY}
    return-url:  https://yourapp.com/api/payment/khalti/callback
    website-url: https://yourapp.com
    sandbox: false                        # ← production mode

  esewa:
    secret-key:   ${ESEWA_SECRET_KEY}
    product-code: ${ESEWA_PRODUCT_CODE}   # ⚠️ real merchant code!
    success-url:  https://yourapp.com/api/payment/esewa/callback
    failure-url:  https://yourapp.com/payment/failed
    sandbox: false                        # ← production mode

Use Only One Gateway

Beans are only created when the corresponding secret-key is present. Configure only what you need:

# Only Khalti — no EsewaClient bean created
nepalpay:
  khalti:
    secret-key:  ${KHALTI_SECRET_KEY}
    return-url:  ${KHALTI_RETURN_URL}
    website-url: ${YOUR_WEBSITE_URL}
