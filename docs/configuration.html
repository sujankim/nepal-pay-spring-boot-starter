Configuration

All nepalpay.* properties and environment variable recommendations.

Khalti Properties

Property Required Default Description
nepalpay.khalti.secret-key ✅ — Khalti secret key. Creates KhaltiClient bean when present.
nepalpay.khalti.return-url ✅ — URL Khalti redirects to after payment.
nepalpay.khalti.website-url ✅ — Your merchant website URL.
nepalpay.khalti.sandbox No true true = dev.khalti.com / false = khalti.com
nepalpay.khalti.timeout-seconds No 10 HTTP timeout for Khalti API calls.

eSewa Properties

Property Required Default Description
nepalpay.esewa.secret-key ✅ — eSewa HMAC secret. Creates EsewaClient bean when present.
nepalpay.esewa.product-code ✅ — Merchant code. Sandbox: EPAYTEST. Production: your code.
nepalpay.esewa.success-url ✅ — URL eSewa redirects to on successful payment.
nepalpay.esewa.failure-url ✅ — URL eSewa redirects to on failed payment.
nepalpay.esewa.sandbox No true true = sandbox / false = production

Sandbox Configuration

nepalpay:
  khalti:
    secret-key:  ${KHALTI_SECRET_KEY}   # test_secret_key_xxx
    return-url:  http://localhost:8080/api/payment/khalti/callback
    website-url: http://localhost:4200
    sandbox: true

  esewa:
    secret-key:   ${ESEWA_SECRET_KEY}   # 8gBm/:&EnhH.1/q
    product-code: EPAYTEST
    success-url:  http://localhost:8080/api/payment/esewa/callback
    failure-url:  http://localhost:4200/payment/failed
    sandbox: true

Production Configuration

nepalpay:
  khalti:
    secret-key:  ${KHALTI_SECRET_KEY}
    return-url:  https://yourapp.com/api/payment/khalti/callback
    website-url: https://yourapp.com
    sandbox: false              # ← production!

  esewa:
    secret-key:   ${ESEWA_SECRET_KEY}
    product-code: ${ESEWA_PRODUCT_CODE}   # ⚠️ must be real merchant code!
    success-url:  https://yourapp.com/api/payment/esewa/callback
    failure-url:  https://yourapp.com/payment/failed
    sandbox: false              # ← production!

Override Beans

Every bean uses @ConditionalOnMissingBean. Define your own bean and NepalPay steps aside: 

@Configuration
public class MyConfig {

    @Bean
    public KhaltiClient customKhaltiClient(
            NepalPayProperties props,
            RestClient.Builder builder) {
        // Your custom setup
        return new KhaltiClient(props.khalti(), builder);
    }
}
