Requirements

⚠️

Merchant registration required. ConnectIPS requires registration with NCHL before use. You cannot test without NCHL credentials. Contact connectips@nchl.com.np.

From NCHL you will receive:

  • Merchant ID (integer)
  • Application ID and Name
  • Application Password
  • CREDITOR.pfx certificate file
  • Password for the .pfx file
🔴

Never commit CREDITOR.pfx to Git. Add *.pfx to your .gitignore immediately. The .pfx contains your RSA private key.

How ConnectIPS Works

Your Backend: buildFormPayload() → RSA-SHA256 sign with CREDITOR.pfx private key → returns ConnectIpsFormPayload ↓ Frontend: POST form to payload.formActionUrl() ↓ User completes bank transfer on ConnectIPS ↓ ConnectIPS redirects to your callback URL ↓ ⚠️ NEVER trust this redirect alone! ↓ Your Backend: validateTransaction(txnId, referenceId, txnAmtPaisa) → POST to ConnectIPS validate API with HTTP Basic Auth → response: { status: "SUCCESS" } ↓ ✅ mark order as paid
ℹ️

The TOKEN field contains the RSA-SHA256 digital signature generated from your CREDITOR.pfx private key. The signature is computed on your server and sent to the frontend as a form value — only the CREDITOR.pfx private key never leaves your server.

Configure

nepalpay:
  connectips:
    merchant-id:  ${CONNECTIPS_MERCHANT_ID}    # e.g. 550
    app-id:       ${CONNECTIPS_APP_ID}          # e.g. "MER-550-APP-1"
    app-name:     ${CONNECTIPS_APP_NAME}
    app-password: ${CONNECTIPS_APP_PASSWORD}
    pfx-path:     ${CONNECTIPS_PFX_PATH}        # file:/app/CREDITOR.pfx
    pfx-password: ${CONNECTIPS_PFX_PASSWORD}
    sandbox: true

pfx-path formats

Format Example Use case
file: file:/app/CREDITOR.pfx Absolute path on server
classpath: classpath:CREDITOR.pfx Inside JAR (not recommended for production)
⚠️

Always load the .pfx from an environment variable path, not hardcoded. Never place the .pfx file inside your source repository.

buildFormPayload()

String txnId = "TXN-" + orderId + "-" + System.currentTimeMillis();
orderRepo.saveTxnId(orderId, txnId); // save BEFORE returning!

ConnectIpsFormPayload payload = connectIpsClient.buildFormPayload(
    ConnectIpsPaymentRequest.builder()
        .txnId(txnId)
        .amountNPR(100L)     // ← auto-converts NPR to paisa (×100)
        .referenceId(orderId)
        .remarks("Payment for order " + orderId)
        .particulars("MyApp")
        .build()
);

// Return payload to frontend
⚠️

Use .amountNPR(100L) for automatic conversion, or .txnAmtPaisa(10000L) if you already have paisa. ConnectIPS uses paisa internally — NPR 100 = 10000 paisa.

Request builder fields

Method Required Description
.txnId(String) Required Unique transaction ID — store in DB first
.amountNPR(long) Required Amount in NPR — auto-converts to paisa × 100
.txnAmtPaisa(long) Required Alternative — amount directly in paisa
.referenceId(String) Required Your order or reference ID
.remarks(String) Optional Shown to user on ConnectIPS gateway
.particulars(String) Optional Additional particulars field

Frontend Form Submission

ConnectIPS uses UPPERCASE field names in the form POST. Submit all fields from the payload: 

// Angular / TypeScript
initiateConnectIps(payload: ConnectIpsFormPayload): void {
    const form = document.createElement('form');
    form.method = 'POST';
    form.action = payload.form_action_url;

    // Note: ConnectIPS field names are UPPERCASE
    const fields = [
        'MERCHANTID', 'APPID', 'APPNAME', 'TXNID',
        'TXNDATE', 'TXNCRNCY', 'TXNAMT', 'REFERENCEID',
        'REMARKS', 'PARTICULARS', 'TOKEN'
    ];

    fields.forEach(field => {
        const input = document.createElement('input');
        input.type  = 'hidden';
        input.name  = field;
        input.value = (payload as any)[field.toLowerCase()] ?? '';
        form.appendChild(input);
    });

    document.body.appendChild(form);
    form.submit();
}
ℹ️

The TOKEN field contains the RSA-SHA256 digital signature generated from your CREDITOR.pfx private key. It is generated on your server — never exposed to the frontend.

validateTransaction()

Called after ConnectIPS redirects back to your callback URL. Always call this — never trust the redirect. 

@GetMapping("/connectips/callback")
public ResponseEntity<String> callback(
        @RequestParam String txnId,
        @RequestParam String referenceId,
        @RequestParam long txnAmt) {

    // ✅ Always validate server-side
    ConnectIpsValidateResponse res =
        connectIpsClient.validateTransaction(txnId, referenceId, txnAmt);

    if (!res.isPaymentSuccessful()) {
        return ResponseEntity.badRequest()
            .body("Not confirmed: " + res.statusDesc());
    }

    orderRepo.markPaid(referenceId);
    return ResponseEntity.ok("Payment confirmed");
}

Response fields

Method Type Description
isPaymentSuccessful() boolean true only when status is SUCCESS
status() String SUCCESS or FAILED
statusDesc() String Human-readable status description
referenceId() String Your original reference/order ID echoed back
txnAmt() String Amount transacted
merchantId() Integer Your NCHL merchant ID echoed back

Also available — test-friendly overload

// Skips RSA signing — useful in tests with mock token
ConnectIpsValidateResponse res =
    connectIpsClient.validateTransactionWithToken(
        referenceId, txnAmtPaisa, "MOCK_TOKEN"
    );

Security Rules

🔴

Never commit CREDITOR.pfx to Git. Add *.pfx to .gitignore. Load it via environment variable.

🔴

Never trust the callback redirect. Always call validateTransaction() server-side.

✅

The RSA token is generated entirely on your server. It never reaches the frontend. Only the signed token value is sent in the form field.

✅

NepalPay caches the RSA private key at startup — no KeyStore loading on every payment. If the .pfx is invalid, the app fails fast at startup with a clear error.

Reactive Usage

Use ConnectIpsReactiveClient from nepal-pay-spring-boot-reactive-starter. buildFormPayload() is synchronous. validateTransaction() returns Mono with both validation and RSA signing wrapped inside Mono.defer().

Build form payload — synchronous

// Synchronous — RSA signing, no HTTP
public ConnectIpsFormPayload startConnectIps(
        String orderId, long amountNPR) {

    String txnId = "TXN-" + orderId + "-"
            + System.currentTimeMillis();
    orderRepo.saveTxnId(orderId, txnId); // save first!

    return connectIpsReactiveClient.buildFormPayload(
        ConnectIpsPaymentRequest.builder()
            .txnId(txnId)
            .amountNPR(amountNPR)
            .referenceId(orderId)
            .build()
    );
}

Validate transaction — reactive

@GetMapping("/connectips/callback")
public Mono<ResponseEntity<String>> callback(
        @RequestParam String txnId,
        @RequestParam String referenceId,
        @RequestParam long txnAmt) {

    // Both validation and RSA signing are inside Mono.defer()
    // so all errors are emitted as Mono.error() signals
    return connectIpsReactiveClient
        .validateTransaction(txnId, referenceId, txnAmt)
        .flatMap(res -> {
            if (!res.isPaymentSuccessful()) {
                return Mono.just(ResponseEntity.badRequest()
                    .body("Not confirmed: " + res.statusDesc()));
            }
            return orderRepo.markPaid(referenceId)
                .thenReturn(ResponseEntity.ok("Payment confirmed"));
        });
}
ℹ️

In the reactive client, input validation and RSA signing are both inside Mono.defer() — any ConnectIpsException is emitted as a reactive error signal, never thrown outside the pipeline.

How ConnectIPS Differs from Other Gateways

Feature Khalti eSewa ConnectIPS
Flow API-first Form POST Form POST
Signature API Key header HMAC-SHA256 RSA-SHA256 (.pfx)
Amount unit Paisa NPR Paisa
Verify auth Bearer key No auth HTTP Basic Auth
Registration Self-service Self-service Via NCHL/bank
Sandbox Self-service Self-service NCHL UAT only
